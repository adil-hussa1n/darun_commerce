# UK Store — Skincare & Beauty Inventory Management Dashboard

A lightweight, modern, and highly responsive temporary inventory management system built for **UK Store** (specializing in skincare, body care, and hair care). 

The application utilizes **Google Sheets** as its primary cloud database via the **SheetDB API** for complete CRUD and transaction operations. It also features a zero-config, interactive **Local Fallback Mode** that uses `localStorage` prefilled with beautiful dummy products if no database endpoint is supplied, facilitating instant previewing and staging.

---

## Key Features

- **Modern & Premium Beauty Aesthetic**: Curated color palette (Taupe, Sage, Blush, Cream) suitable for beauty products, dynamic card micro-animations, and smooth page transitions.
- **Dynamic Live Preview Card**: Adding products shows a live, rendered beauty card preview matching inputs (including direct image URL loading).
- **Interactive Sales Interface**: Cards support search matching, instant category tab filtering, real-time stock checks, adjustable cart quantities, and celebratory screen confetti upon checkout.
- **Stock Warnings**: Automatic warnings (amber alerts) trigger for products with `< 5` units in stock, and red alerts trigger for out-of-stock items.
- **Google Sheets Integrations**: Complete CRUD operations via SheetDB, matching stock changes and recording sales logs in separate spreadsheets.
- **Vercel Deploy Ready**: Configured with a `vercel.json` router fallback to prevent SPA routing 404s.

## Database Configuration (Google Sheets)

To hook up your own Google Sheet as the database:

### 1. Create your Google Spreadsheet
Create a new Google Sheet (e.g. named `UK Store Database`) and add **two individual sheets (tabs)** at the bottom named exactly:
1. `products`
2. `sales`

### 2. Configure Sheet Columns (Headers)
In the very first row of each sheet, configure the column names exactly as follows:

#### Sheet 1: `products`
| id | name | category | buy_price | sell_price | stock | image | notes | created_at |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- |

*   **id**: Unique identifier (auto-generated by the application)
*   **name**: Product Name
*   **category**: `Skin Care`, `Body Care`, or `Hair Care`
*   **buy_price**: Original purchase cost (decimal number)
*   **sell_price**: Retailing price (decimal number)
*   **stock**: Inventory count (integer number)
*   **image**: Direct image URL path
*   **notes**: Detailed descriptions or notes
*   **created_at**: Product insertion timestamp

#### Sheet 2: `sales`
| sale_id | product_name | category | quantity | unit_price | total_price | date |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- |

*   **sale_id**: Transaction ID (auto-generated by the application)
*   **product_name**: Name of the sold item
*   **category**: Category of the item
*   **quantity**: Quantity sold (integer)
*   **unit_price**: Retailing price of a single unit
*   **total_price**: Quantity × Unit Price
*   **date**: Transaction date and time

### 3. Generate SheetDB API URL
1. Go to [SheetDB.io](https://sheetdb.io) and sign up/in.
2. Click **Create New API** and paste the shareable link of your Google Spreadsheet (make sure your sheet sharing permissions allow edits, e.g. "Anyone with the link can edit", or connect your Google Account).
3. Copy your API URL (e.g. `https://sheetdb.io/api/v1/58f61xxxxxxxx`).

## Local Setup Instructions

1.  **Clone / Download the files** to your project folder.
2.  **Install dependencies**:
    ```bash
    npm install
    ```
3.  **Configure environment variables**:
    Copy the `.env.example` file and rename it to `.env`:
    ```bash
    cp .env.example .env
    ```
    Add your SheetDB API URL:
    ```env
    VITE_SHEETDB_API_URL=https://sheetdb.io/api/v1/YOUR_API_ID
    ```
    *Note: If `VITE_SHEETDB_API_URL` is blank or missing, the application automatically runs in Local Fallback Mode, enabling full adding, listing, updating, and selling simulation using mock data stored in `localStorage`.*
4.  **Run development server**:
    ```bash
    npm run dev
    ```
    Open [http://localhost:5173](http://localhost:5173) in your browser.

## Vercel Deployment

Deploying the UK Store dashboard to Vercel takes only a minute:

1.  Push your code to a Git repository (GitHub, GitLab, Bitbucket).
2.  Import the repository into **Vercel**.
3.  In the Vercel project **Settings**:
    - Under **Environment Variables**, add `VITE_SHEETDB_API_URL` and paste your SheetDB API URL.
4.  Click **Deploy**.
5.  *Note: The included `vercel.json` file handles all client-side URL routing, ensuring that refreshing pages like `/sell-products` will not trigger a 404.*
